## 1. 产品概述

### 1.1 一句话描述

以**腾讯**为背景的 **AI 赋能 HR 单人模拟闯关游戏**：玩家先按真实组织架构选择事业群、产品小组与职级，再选定 HR 角色类型（沟通/创意/数据/技术），在「工作日」节奏下完成来自 Task 题库的 HR 挑战；通关后升级职级、解锁装备与更复杂场景。

### 1.2 产品愿景

让校招/实习生/HR 新人在**低风险、高拟真**环境中体验「AI + HR」的真实工作流，理解四型 HR 的能力模型，建立对腾讯 S3 职能线（HR 与管理线）及六大事业群协作关系的认知。

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **拟真但不泄密** | 组织架构、岗位名称参考公开信息；业务流程为教学化抽象，不使用真实内部系统与数据 |
| **AI 是工具不是外挂** | 挑战允许/鼓励使用 AI 工具，但评分考察「人机协作质量」而非单纯产出速度 |
| **角色差异化** | 四类 HR 的 Task 类型、装备、评分维度显著不同 |
| **单人优先** | v0.1 仅单人；存档本地或账号级，无 PvP |
| **短局可玩** | 单局 15–30 分钟可完成「入职前 7 个工作日」主线 |

## 2. 背景与目标

### 2.1 业务背景

腾讯 S3 职能系统—HR 与管理线推出 **AI-HR 培训生（HR Star）** 项目，将岗位细分为四大方向（见官方 JD 与 TCamp 介绍视频）：

- **沟通型**：招聘调配、员工培训、HRBP、员工关系
- **创意型**：企业文化、雇主品牌（活动策划/传播/内容制作）
- **分析型**：人力规划、OD、人才发展、薪酬/绩效管理
- **技术&应用型**：HR 数字化产品、AI 应用科学家、数据工程、后台开发

项目 slogan：**在 AI 时代寻找 π 型人才**——既懂 HR 业务，又掌握 AI 工具或技术思维。

### 2.2 产品目标

| 目标类型 | 指标（MVP） |
|----------|-------------|
| **认知** | 玩家能说出四型 HR 的核心差异及 2 个以上对应业务场景 |
| **参与** | 单次会话完成 ≥3 天挑战的比例 ≥60% |
| **完成** | 7 日主线通关率 ≥30%（可调难度） |
| **传播** | 支持分享「职级卡 / 通关海报」（v0.2） |

### 2.3 非目标（v0.1 不做）

- 多人在线、排行榜、公会
- 真实对接 join.qq.com 或内部 HRIS
- 完整覆盖 HR 三支柱（COE/HRBP/SSC）全模块
- 付费内购与商业化

## 4. 世界观与组织设定

### 4.1 顶层架构

```
腾讯公司
├── 六大事业群（玩家服务 BG，挑战任务来自各 BG 的 HR 需求）
│   ├── WXG  微信事业群
│   ├── PCG  平台与内容事业群
│   ├── IEG  互动娱乐事业群
│   ├── TEG  技术工程事业群
│   ├── CSIG 云与智慧产业事业群
│   └── CDG  企业发展事业群
│
└── S3 职能系统 — HR 与管理线（玩家所属职能线，不可选其他职能）
    ├── COE  专家中心（政策、OD、薪酬等）
    ├── HRBP 业务合作伙伴（嵌入各 BG）
    └── SSC  共享服务中心（流程、系统、员工服务）
```

**玩家身份**：S3 下派驻某一 **事业群** 的 **产品/业务小组** 的 HR（培训生/实习生起），非 BG 业务研发岗。

### 4.2 事业群与产品小组（MVP 配置）

每个 BG 配置 **2–3 个产品小组**，用于 Task 文案中的「需求方」占位（如「PCG · 腾讯视频」「WXG · 微信支付」）。

| 事业群 | 产品小组示例 | HR 协作高频场景 |
|--------|--------------|-----------------|
| WXG | 微信基础、微信支付、企业微信 | 大规模员工沟通、合规培训、技术岗招聘 |
| PCG | 腾讯视频、QQ、腾讯新闻 | 雇主品牌活动、内容型校招、工作室文化 |
| IEG | 王者荣耀、和平精英、电竞 | 快节奏招聘、赛事活动、创意文化 |
| TEG | 基础架构、安全、AI 平台 | HR 数字化、技术型 HR 协作、数据安全 |
| CSIG | 腾讯云、智慧产业 | 政企客户活动、解决方案团队 OD |
| CDG | 腾讯广告、理财通 | 商业化团队激励、广告业雇主品牌 |

### 4.3 职级体系

| 职级代码 | 名称 | 解锁条件 | 游戏内权益 |
|----------|------|----------|------------|
| L0 | **实习生** | 初始可选 | 基础 Task、导师提示 3 次/天 |
| L1 | **培训生** | 完成 Day1–3 | 解锁「装备栏」第 2 格 |
| L2 | **助理 HR** | 完成 Day4–5 | 挑战限时 +20%、AI 工具栏 +1 |
| L3 | **HR 专员** | 完成 Day6–7 | 解锁「困难模式」Task |
| L4+ | **高级 HR**（v0.2） | 二周目 | 跨 BG 协作 Task |

**说明**：职级名称对齐腾讯 HR 语境，数值为游戏抽象，不对应真实职级体系。

## 5. 角色体系（四类 HR）

### 5.1 角色总览

| 角色 ID | 名称 | Slogan | 需求热度 | 潜在岗位方向 |
|---------|------|--------|----------|--------------|
| `comm` | **沟通型 HR** | 用 AI 助力人与业务的链接 | 需求多多 | 招聘调配、员工培训、HRBP、员工关系 |
| `creative` | **创意型 HR** | 用 AIGC 释放创意 | — | 企业文化、雇主品牌、活动策划/传播/内容 |
| `analyst` | **分析型 HR** | 用 AI 洞悉数据真相 | 需求多多 | 人力规划、OD、人才发展、薪酬管理 |
| `tech` | **技术型 HR** | 用技术重塑管理 | — | HR 数字化 PM、AI 应用科学家、数据工程、后台开发 |

### 5.2 各角色「通关指南」与「必备装备」

> **通关指南**：该角色在模拟场中的能力成长路径与评分侧重。  
> **必备装备**：游戏内可解锁/携带的 AI 工具与能力卡，影响 Task 可选策略与得分上限。

---

#### 5.2.1 沟通型 HR（Communication）

**核心能力模型**

| 维度 | 描述 |
|------|------|
| 人际感知力 | 识别干系人情绪、诉求与冲突点 |
| 问题解决 | 在约束下给出可执行的沟通方案 |
| 沟通协作 | 结构化表达、会议 facilitation |
| 逻辑思维 | 需求澄清、优先级排序 |
| AI 学习意愿 | 使用 AI 辅助话术、纪要、翻译 |

**通关指南（玩家可见）**

1. **Day 1–2**：掌握「需求澄清四步法」— 听诉求 → 问背景 → 对齐目标 → 确认下一步  
2. **Day 3–4**：学会用 AI 生成沟通草稿，但**必须人工改写**关键句（评分项）  
3. **Day 5–7**：处理跨 BG 冲突型 Task，考察利益平衡与向上汇报  

**必备装备（初始 + 解锁）**

| 装备 | 类型 | 效果 |
|------|------|------|
| 🎧 **倾听者徽章** | 初始 | 冲突 Task 中显示「情绪标签」提示 |
| ✍️ **AI 话术助手** | Day2 解锁 | 生成 3 版沟通邮件/IM 草稿 |
| 📋 **结构化纪要模板** | Day4 解锁 | 会议类 Task 自动提纲 |
| 🤝 **HRBP 手册** | Day6 解锁 | 显示业务方 KPI 背景卡片 |

**典型 Task 类型**：候选人拒 Offer 沟通、业务经理催招、员工关系投诉初筛、培训需求访谈。

---

#### 5.2.2 创意型 HR（Creative）

**核心能力模型**

| 维度 | 描述 |
|------|------|
| 想象力与审美 | 活动/物料视觉与叙事一致性 |
| 内容创作力 | 文案、短视频脚本、传播节奏 |
| AIGC 能力 | AI 绘图、AI 写作、AI 视频 |

**通关指南**

1. **Day 1–2**：熟悉品牌基调（腾讯蓝、年轻化、科技感）与合规红线（无夸大、无歧视）  
2. **Day 3–5**：限时产出类 Task 为主 — 海报 / 短文案 / 9 图 / 30s 脚本  
3. **Day 6–7**：大型雇主品牌 Campaign 分阶段交付（Brief → 草案 → 终稿）  

**必备装备**

| 装备 | 类型 | 效果 |
|------|------|------|
| 🎨 **AIGC 绘图台** | 初始 | 海报 Task 可生成 2 个视觉草案 |
| 📝 **AI 写作台** | 初始 |  slogan / 推文 / 活动规则 |
| 🎬 **AI 视频脚本器** | Day3 解锁 | 输出分镜 + 旁白 |
| 📐 **品牌规范包** | Day5 解锁 | 自动校验字体/色值/禁用词 |

**典型 Task 类型**（含用户举例）：

> **Day 3 示例**：「请为 **腾讯 PCG · 腾讯视频** 的 *2026 校招空宣活动* 设计一张主视觉海报，要求体现『年轻、多元、AI 赋能』，限时 **12 分钟**，需提交：主标题 + 副标题 + 画面描述（或上传图片）。」

#### 5.2.3 分析型 HR（Analyst / Data）

**核心能力模型**

| 维度 | 描述 |
|------|------|
| 逻辑与洞察 | 从数据中发现问题而非堆砌图表 |
| 人性研究兴趣 | 结合组织行为解释数据异常 |
| AI 分析工具 | 建模、预测、可视化、归因 |

**通关指南**

1. **Day 1–2**：读懂 HR 核心指标（编制率、流失率、敬业度、招聘漏斗）  
2. **Day 3–5**：在脏数据 / 样本不足下做「可辩护的结论」  
3. **Day 6–7**：输出 OD 或薪酬方向的决策建议 Memo  

**必备装备**

| 装备 | 类型 | 效果 |
|------|------|------|
| 📊 **HR 指标仪表盘** | 初始 | Task 内置数据集可视化 |
| 🧮 **AI 分析助手** | 初始 | 回归/分组/趋势一句话解读 |
| 🔮 **预测沙盒** | Day4 解锁 | 流失/编制预测简化模型 |
| 📑 **Memo 生成器** | Day6 解锁 | 结构化输出「发现-建议-风险」 |

**典型 Task 类型**：编制缺口分析、离职归因、敬业度调研解读、薪酬带宽异常排查。

#### 5.2.4 技术型 HR（Tech & Application）

**核心能力模型**

| 维度 | 描述 |
|------|------|
| 技术热情 | 理解 HR 场景的技术实现边界 |
| 创新思维 | 跨界洞察，产品 + 技术 + 人 |
| 探索欲 | 对「人的数据化与智能化」好奇 |

**通关指南**

1. **Day 1–2**：熟悉 HR 数字化触点（招聘 Bot、智能问答、流程机器人）  
2. **Day 3–5**：需求 → 原型/伪代码 → 验收标准  
3. **Day 6–7**：AI Agent 场景设计 + 数据安全/伦理 checklist  

**必备装备**

| 装备 | 类型 | 效果 |

## 6. 核心玩法循环

```
选择组织（BG + 小组 + 职级）
        ↓
选择角色类型（四选一）→ 查看通关指南 & 初始装备
        ↓
进入「工作日」主界面（日历 / 邮件 / Task 面板）
        ↓
每日触发 1 个【主线挑战】+ 0–1 个【支线】
        ↓
阅读 Brief → 使用装备 & AI 工具 → 提交成果
        ↓
系统评分（规则 + AI 辅评）→ 反馈报告
        ↓
   通过 ──→ 经验值 + 可能升级 + 解锁装备 → 下一天
   失败 ──→ 消耗「复盘次数」或重试（降分）→ 同 Day 重做
        ↓
完成 Day7 → 本周通关 → 职级提升 / 生成成长报告
```

### 6.1 时间结构

| 概念 | 规则 |
|------|------|
| **游戏日** | 1 游戏日 = 1 个主线 Task，约 5–15 分钟 |
| **主线** | Day1–Day7 固定有挑战，难度曲线上升 |
| **支线** | 随机触发，奖励装备碎片或导师提示 |
| **限时** | 部分 Task 有倒计时；超时仍可提交，得分 ×0.7 |

### 6.2 胜负条件

| 结果 | 条件 |
|------|------|
| **通关** | 单 Task 得分 ≥ 60（百分制） |
| **优秀** | 得分 ≥ 85，额外 EXP + 成就 |
| **失败** | 得分 < 60，允许重试 1 次（同 Day） |
| **一周通关** | Day1–7 至少 5 天主线的「通关」 |

## 8. 评分与 AI 赋能

### 8.1 评分架构

```
最终得分 = 规则引擎分（40%）+ AI 辅评（40%）+ 过程分（20%）
```

| 模块 | 说明 |
|------|------|
| **规则引擎** | 必填项、字数、时限、禁用词、格式 |
| **AI 辅评** | LLM 按 rubric 维度打分 + 文字反馈（需 prompt 模板化） |
| **过程分** | 是否使用装备、是否查看 Brief、重试次数 |

### 8.2 AI 赋能点（产品内）

| 场景 | AI 能力 | 边界 |
|------|---------|------|
| Task 生成 | 根据 BG/小组/Day 动态组装 Brief | 不生成真实内部政策 |
| 辅评 | 多维度评分 + 改进建议 | 需人工抽检校准 |
| 导师 NPC | 卡关时 3 次/天提示 | 不直接给满分答案 |
| 创意类 | 接入绘图/写作 API（或模拟） | MVP 可用文本描述代替真实出图 |
| 沟通类 | 模拟业务方/员工多轮对话 | 预设人格与冲突脚本 |

### 8.3 合规与伦理（产品层）

- 禁止收集真实员工 PII；所有人物/数据为虚构  
- Task 文案标注「模拟实训，非腾讯官方」  
- AI 输出需过**敏感词 + 歧视/偏见**规则过滤  
- 技术型 Task 强调数据权限与算法公平性 checklist
