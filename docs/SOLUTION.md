# 腾讯 AI 赋能 HR 模拟场 · 方案说明

## 一、问题诊断

**目标用户**：尚未进入职场、希望了解 HR 日常工作的校招求职生，以及初入职场、有成长意愿但对业务缺乏体感的新人。他们普遍「知道 HR 很重要，却不知道 HR 每天在做什么」——缺少可反复练习、低试错成本的场景化训练。

**核心痛点**：

1. **认知断层**：招聘宣讲、课程视频多为概念介绍，难以建立「接到需求 → 拆解任务 → 人机协作交付 → 被评估反馈」的完整工作流感知。
2. **场景缺失**：真实 HR 工作高度情境化（不同 BG、不同角色、不同干系人语气），纯阅读或刷题无法模拟「今天业务方催 HC、明天要做校招海报」的节奏。
3. **AI 协作能力空白**：雇主品牌、沟通回复等岗位 increasingly 依赖 AI 辅助，但新人缺少在任务压力下练习 Prompt 写作、迭代与人工改写的场域。

**场景选择**：聚焦 **雇主品牌 / 校招传播（创意型 HR）** 与 **业务沟通 / 员工关系（沟通型 HR）** 两条培训生主线，以腾讯 S3 HR 线为叙事壳，降低陌生感的同时保持可迁移的方法论。

---

## 二、方案设计

**产品定位**：一款 **单人、浏览器可玩的 HR 实训闯关游戏**，用「7 日征途」替代传统课件，让玩家以培训生身份完成每日 Brief，在限时内交付成果并获得双维反馈。

**关键设计**：

| 模块 | 设计要点 |
|------|----------|
| 入职引导 | 三级组织选择（事业群 → 小组 → 培训生）+ 角色选择 + 卷轴式通关指南与七日路线地图 |
| 主控制台 | 日历进度、邮件式 Task Brief、装备与导师提示，强化「上班感」 |
| 挑战页 | 创意型：平台内 AI Chat 完成 AIGC 文案/海报；沟通型：AI 话术草稿 + 人工改写提交 |
| 评分反馈 | 创意任务 **成稿分 + 提示词分** 分开评价；沟通任务强调人工改写度 |
| 成长闭环 | Day7 成长报告、能力雷达、关卡复盘，支持 localStorage 断点续玩 |

**为何用「通关游戏」**：将抽象能力拆解为 7 个可感知里程碑（Slogan → 9 宫格 → 主视觉海报 → …），每关有明确交付物与通关线（≥60 分），满足新人「想进步、需要正反馈」的心理；游戏化降低对 HR 专业门槛的畏惧，把学习动机从「背知识」转为「过关」。

## 三、AI 工具选型与关键配置

| 能力 | 选型 | 理由 |
|------|------|------|
| 对话 / 辅评 | 腾讯混元（OpenAI 兼容接口） | 与目标场景（腾讯 HR）一致；可降级为 mock 保证演示可用 |
| AIGC 出图 | 混元 HY-Image（TokenHub） | 平台内完成海报任务，无需跳转外部工具 |
| 评分 | 规则引擎 + LLM 结构化 JSON | 可解释、可标定；Prompt 维单独评估迭代行为 |
| 开发 | Cursor + React/Vite + FastAPI | 快速迭代 UI（卷轴地图、水墨动效）与 API |

**关键配置**：`LLM_PROVIDER` / `IMAGE_PROVIDER` 环境变量切换 mock 与生产；Task 级 `system_prompt_template` 注入 Brief 与品牌约束；Chat 历史持久化用于 Prompt 评分。

## 四、迭代与效果

**迭代记录**：

1. v0：静态 PRD → 可玩 MVP（双角色 × 7 日）
2. UI：羊皮卷轴指南、七日水墨路线地图、Mentor 鹅 Tips、Prompt 雷达图
3. 评分：沟通型分支对话树、管理看板聚合匿名提交数据

**效果评估（预期）**：

- 完播 Day1 ≥65%、Day3（含 AIGC）≥45%，验证场景吸引力
- Prompt 分随多轮迭代上升，验证「过程评分」教学价值
- 用户反馈：从「不知道 HR 做什么」到「能描述一次完整 Task 流程」

**局限与后续**：MVP 为本地存档、mock 可独立演示；接入混元后可体验真实 AIGC；分析型/技术型 HR 路径待扩展。
